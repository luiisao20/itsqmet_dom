DOM

Document Object Model, introducción al DOM.

El modelo de objetos del documento DOM por sus siglas en inglés es una interfaz de programación del lenguaje de tipado HTML. Este define la estructura lógica de los documentos y la forma que se accede a ellos, pero no especifica relación alguna entre ellos. Proporciona propiedades y métodos para acceder a los objetos que lo componen y modificarlos a nuestro antojo.

Métodos y valores del DOM

Los métodos son acciones que puedes realizar en elementos HTML. Las propiedades son valores que se pueden configurar o cambiar

Método Sintaxis Descripción
Buscar elemento por ID document.getElementById("id") Devuelve un elemento del DOM que tenga el id seleccionado
Buscar elemento por etiqueta document.getElementByTagName("p") Devuelve un array de elementos que contengan la etiqueta seleccionada
Buscar elementos por el nombre de clases document.getElementByClassName("intro") Regresa un listado de elementos que contengan el nombre de la clase
Buscar elementos por selectores document.querySelectorAll("p.intro") Regresa un listado de elementos que encajan con la descripción de búsqueda
Obtener elementos hijos .children Devuelve una lista de elementos HTML hijo
Buscar el elemento padre .parentElement Devuelve el padre de un elemento HTML
Obtener el texto .textContent Devuelve el texto de un elemento y también lo puede cambiar
Obtener el texto renderizado .innerText Devuelve el texto de un elemento y también lo puede cambiar (afectando los estilos CSS)
Obtener el contenido HTML .innerHTML Devuelve el contenido HTML de un elemento y también lo puede cambiar

Tipos de DOM

El DOM Real es la forma en la que está estructurada la página HTML que se puede visualizar en cualquier navegador, seleccionando Inspeccionar y abriendo la pestaña de elementos

JavaScript nos proporciona varias funciones (descritas anteriormente) para acceder a estos elementos, modificarlos o eliminarlos

El DOM virtual es una representación ligera y artificial del DOM real. Desde la aparición de diferentes Frameworks del ecosistema de JavaScript, el concepto de DOM virtual tomó forma, esta no es más que una copia original del DOM donde se gestionan los cambios y actualizaciones para posteriormente introducrilos al DOM real.

El trabajo en este DOM acelera los cambios y sólo afecta al nodo seleccionado, sin afectar a todo el documento, lo que ayuda a una mejor gestión del mismo, aunque posee ligeras desventajas como peor rendimiento, más coste de memoria y mayor complejidad

El Shadow DOM o sombra del DOM es un concepto moderno para crear elementos HTML en el DOM teniendo su propio DOM en particular, por lo tanto siendo independientes del resto del documento facilitando su encapsulación.

Si a un DOM normal se le agrega un Shadow DOM a un elemento HTML pasaríamos a tener un DOM con su propio código HTML, lo que ayudaría a hacer cambios sólo en ese elemento sin afectar al resto del documento.

Este concepto retoma gran relevancia cuando se empieza a trabajar con componentes reutilizables de forma nativa, lo que facilita al programador la organización de su propio código.

El concepto de Light DOM tiene sentido con un elemento que posea Shadow DOM. Cuando creamos un Shadow DOM para un elemento, este oculta el contenido que ya tenía el elemento. El contenido oculto es el denominado informalmente como Light DOM.
